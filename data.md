#### Estructura Recomendada del Portafolio
1. Hero Section (Presentación Impactante)

Es el primer contacto visual donde tu fondo de "light leaks" violetas debe brillar.

    Contenido: Nombre, rol y una frase que resuma tu stack actual.

    Skill recomendada: hero-generator para definir el espaciado y los botones de acción iniciales.

2. Timeline Educativo (Línea de Tiempo)

En lugar de una sección de Testing, usaremos esta línea de tiempo para mostrar tu trayectoria desde que iniciaste en la programación hasta tus proyectos actuales como "Pocket" y la App de Restaurante.

    Hitos sugeridos:

        Inicio: Primer acercamiento al código.

        Especialización: Cuando te enfocaste en Java y el ecosistema Spring.

        Actualidad: Tu búsqueda activa de empleo y desarrollo de aplicaciones full-stack.

    Skill recomendada: timeline-builder o vertical-list-generator. Pídele que use nodos circulares con el color Plum (#4B1D3F) para mantener la coherencia visual.

3. Stack Tecnológico (Skills)

Lista tus herramientas técnicas divididas por categorías para facilitar la lectura a los reclutadores.

    Contenido: Java (Core y Spring Boot), Angular, PostgreSQL/Supabase, y herramientas de despliegue como Vercel o AWS.

4. Proyectos Seleccionados

Tarjetas visuales que enlacen a tus repositorios de GitHub.

    Pocket: Tu aplicación de gestión financiera con autenticación.

    App de Restaurante: El sistema con base de datos Supabase y gamificación.

    Skill recomendada: card-grid-scaffold con efectos de Glassmorphism para que el fondo violeta sea sutilmente visible tras las tarjetas.

5. Sobre mí y Contacto

    Contenido: Un resumen de tus objetivos profesionales como desarrollador junior y enlaces a tus redes profesionales (LinkedIn y GitHub)..

#### Estilo Visual: Protocolo MARATHON

1. Paleta de Colores Evolucionada

    Fondo: Negro Profundo (#000000).

    Primario (Neon Violet): #640dad.

    Secundario (Neon Indigo): #7B00FF.

    Acento de Alerta (Marathon Red/Orange): #FF3E00 (Este toque de naranja/rojo vibrante es icónico de Marathon para resaltar elementos críticos).

    Superficies: Gris Oxigenado Muy Oscuro (#0A0A0A) con bordes ultra finos.

2. Estructura de UI (UI de "Runner")

    Data-Heavy Layout: En lugar de bloques simples, usa bordes que no cierran del todo (esquinas en ángulo de 45° o líneas truncadas).

    Tipografía: Usa una fuente Sans-Serif geométrica muy técnica (tipo Oswald o Michroma) para los títulos y Monoespaciada (JetBrains Mono) para los datos técnicos de tu stack de Java y Spring.

    HUD Overlays: Añade elementos decorativos en las esquinas de la pantalla como "SYSTEM STATUS: NOMINAL", "USER: F. AIETA", o coordenadas de red, simulando la interfaz de un casco.

3. Estilos de Código y Componentes

    Glassmorphism Espectral: Las tarjetas de tus proyectos (Pocket y App de Restaurante) deben ser translúcidas pero con un ligero tinte violeta y un ruido granulado muy fino para dar textura de "pantalla vieja".

    SVG Dinámicos: Los iconos de Docker, AWS y Java deben tener un filtro de "flicker" (parpadeo) muy tenue cuando pasas el ratón, como si la señal estuviera interfiriendo.

4. Micro-interacciones

    Efecto Glitch Controlado: Al navegar entre secciones, aplica una distorsión rápida de color (aberración cromática) que dure apenas 0.1s.

    Carga de Datos: Cuando abras el portafolio, haz que el texto de tu nombre y rol aparezca con un efecto de "escritura de terminal" o con un escaneo vertical de luz.
